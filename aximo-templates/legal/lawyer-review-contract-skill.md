# Навык: review-contract (проверка договора по нашему playbook)

> Прислал юрист Aximo. Это наша подстраховка: здесь зашиты грани, за которые мы **не выходим**. Запускать на любом входящем договоре до подписания.

## Что делает

Берёт договор (PDF/текст), определяет **нашу сторону** (Aximo — как правило Исполнитель/Supplier), идёт по ключевым статьям и по каждой ставит флаг:

- 🟢 **GREEN** — приемлемо, не трогаем.
- 🟡 **YELLOW** — вне стандарта, но в зоне переговоров → даём правку.
- 🔴 **RED** — за красной линией, материальный риск → правим обязательно, при необходимости — к CEO.

## Playbook Aximo — красные линии

1. **Оплата (главное).** Мы **не работаем без предоплаты.** Требуем аванс (≥ 50% при подписании) и/или **поэтапную оплату**. Постоплата «через N дней после полной приёмки» без аванса — 🔴. Net-срок длиннее 30 дней — 🟡/🔴.
2. **Ответственность.** Лимит ответственности = **сумма полученных по договору платежей**. **Неограниченная ответственность — 🔴.**
3. **Штрафы / неустойки.** Только разумные и **с потолком**. Штраф за просрочку выше 5% / кратный стоимости / «по первому требованию» — 🔴. Сроки должны быть **оценочными** (estimates), не жёсткими с карами.
4. **Индемнити (возмещение).** Взаимное и с потолком. Одностороннее неограниченное возмещение в пользу клиента — 🔴.
5. **Интеллектуальная собственность.** Наши **пред-существующие** материалы, инструменты и know-how остаются за нами. Клиенту переходят только созданные Deliverables и **только после полной оплаты**.
6. **Расторжение.** «For convenience» — допустимо при оплате уже выполненного. Одностороннее расторжение в короткий срок без оплаты выполненного — 🟡/🔴.
7. **Прочее.** Эксклюзив / non-compete — осторожно (🟡). DPA при персональных данных — должен быть. Применимое право/юрисдикция — ок, если разумны.

## Как идти

1. Принять договор, кратко зафиксировать: документ, стороны, наша сторона, дедлайн.
2. Постатейно сверить с playbook → проставить флаги.
3. По каждому 🟡/🔴 собрать **редлайн** в формате ниже.
4. Дать краткий итог: топ-проблемы и стратегию.

## Формат редлайна (по каждому 🟡/🔴)

```
Пункт: <ссылка на статью>
Сейчас: "<точная формулировка из договора>"
Правка: "<наша предлагаемая формулировка>"
Почему: <1–2 предложения, можно показать второй стороне>
Приоритет: Must-have / Should-have / Nice-to-have
```

## Формат вывода

- **Итог проверки** — документ, стороны, наша сторона, дедлайн.
- **Ключевые проблемы** — топ-2–3 с флагами.
- **Постатейно** — список статей с 🟢/🟡/🔴.
- **Редлайны** — по проблемным пунктам (формат выше).
- **Рекомендация** — что отправить на согласование, что вынести на CEO.

## Гейты

- Мы **не подписываем** из навыка — готовим правки для согласования. Финальное «да» на нестандартные уступки — у **CEO** (Marina).
- Ничего не выдумывать вне текста договора. Цитаты — точные.
- Все суммы и сроки — как в документе.

# Aximo — модель расчёта проекта (математика сметы)

> Внутренний файл для расчёта смет. По нему навык `proposal` считает стоимость под набор требований клиента. Все суммы в **$**.

## Из чего складывается цена

| Позиция | Что входит | Цена |
|---|---|---|
| **Внедрение** (фикс, один раз) | Аудит процесса, настройка решения, интеграция в системы клиента, обучение команды | **$14 000** |
| **Автоматизация одного потока** | Один тип документа/процесса: распознавание → извлечение полей → правила → тестирование на данных клиента | **$8 000** за поток |
| **Поддержка** (ретейнер, опц.) | Мониторинг, дообучение моделей, сопровождение по SLA | **$2 000 / мес** |

## Как считать

1. **База** = Внедрение ($14 000) — всегда.
2. **+ Потоки** = (число типов документов/процессов) × $8 000.
3. **Итого проект** = База + Потоки.
4. **Поддержка** — отдельной строкой, опционально, помесячно (в «Итого проекта» не входит).

## Этапность (если клиент хочет поэтапно)

- **Этап 1** = Внедрение + первый (приоритетный) поток.
- **Этап 2** = остальные потоки.
- Оплата привязывается к этапам. Скидок за этапность нет — цена та же, просто разбита.

## Правила

- Не выдумывать позиции и цены вне этой таблицы. Если клиент просит что-то нестандартное — пометить `[уточнить у CEO]`, не придумывать сумму.
- Скидки и нестандартные условия — только с согласования CEO (Marina).
- Валюта — $. Числа — с разделителем тысяч (например, $38 000).

## Пример

Клиент хочет автоматизировать 3 типа документов (счета, договоры, акты), поэтапно:
- Внедрение: $14 000
- Потоки: 3 × $8 000 = $24 000
- **Итого проект: $38 000**
- Этап 1 (внедрение + счета): $22 000 · Этап 2 (договоры + акты): $16 000
- Поддержка: $2 000 / мес (опционально, после запуска)

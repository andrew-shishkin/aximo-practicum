# Квалификация входящего лида

## Шаг 1 — взять лид

Возьми имя и корп.почту контакта (из заявки/карточки CRM). Из почты — **домен**. По TLD пойми, российская компания (`.ru`) или зарубежная.

## Шаг 2 — сайт

Зайди на сайт компании по домену: чем занимаются, размер, гео.

Для российских компаний дополнительно найди **ИНН**: подвал сайта, страницы «Контакты»/«Реквизиты»/«Юридическая информация», публичная оферта или политика конфиденциальности. Не нашёл на сайте быстро — загугли «<компания> ИНН» или проверь ЕГРЮЛ. ИНН российской компании всегда публичен.

## Шаг 3 — обогатить (дёшево, без интернет-поиска конкретных ЛПР)

- **Зарубежная компания** — через коннектор (Apollo/Clay/ZoomInfo), если подключён: фирмографика + пара контактов.
- **Российская компания** — скриптом по ИНН (DataNewton/СБИС/Контур.Фокус): выручка, численность, руководитель (кандидат в ЛПР).

Полноценный поиск конкретных ЛПР в интернете — не здесь, а в навыке холодного outbound.

## Шаг 4 — сверить с ICP

Прочитай ICP из бизнес-CLAUDE.md (в практикуме — `aximo-CLAUDE.md`; в реальном проекте — `CLAUDE.md` твоего проекта). Пройдись по критериям: размер, индустрия, гео, повторяющаяся рутина/бюджет — отметь ✓/✗ по каждому.

## Шаг 5 — вердикт и запись

Вынеси вердикт: **КВАЛ** / **не КВАЛ** / **КВАЛ-с-оговоркой** + короткое обоснование + рекомендованный заход.

Запиши в CRM (`aximo-crm/data.js`) **с подтверждения**: обнови карточку компании (домен, индустрия, гео, сайт, ИНН/выручка — если РФ), добавь заметку с вердиктом в сделку, **перенеси стадию**: КВАЛ / КВАЛ-с-оговоркой → `Qualified`, не КВАЛ → `Closed Lost` (с причиной) — лид не должен оставаться в «New». Пересобери доску (`python3 aximo-crm/_build.py`; Windows — `python`; либо `node aximo-crm/_build.js`) и перепроверь, что стадия действительно сменилась, прежде чем отчитаться.

## Формат вывода

1. Вердикт (строкой).
2. Профиль компании.
3. Сигналы по ICP (✓/✗ по каждому критерию).
4. Контакты (если появились новые).
5. Рекомендованный заход.
6. Источники.

## Гейты безопасности

- Запись в CRM — только с подтверждения.
- Ключи — из `.env`, никогда не печатать.
- Ничего необратимого без явного «ок».

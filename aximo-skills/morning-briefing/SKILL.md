# Утренний брифинг

## Шаг 1 — собрать данные

Возьми открытые сделки из доступной CRM:
- в практикуме — `aximo-crm/data.js` (читай молча, не показывай ученику JSON);
- в реальном проекте — из подключённой CRM (коннектор или MCP).

Для каждой сделки нужны: название, сумма, стадия, владелец, дни без активности, следующий шаг, заметки. Для счетов (стадия Invoice) — номер счёта и срок оплаты.

## Шаг 2 — приоритизация по категориям

- 🔥 **Горящие** — продвинутая стадия + недавняя активность + от нас ждут действия.
- ⏳ **Застряли** — активная стадия, 7+ дней без касания.
- 💸 **Просроченные счета** — стадия Invoice, срок оплаты вышел.
- 🤝 **Перешли на тебя** — владелец недоступен (в отпуске) — сейчас на тебе.
- 🌱 **Свежие лиды** — стадия New.

Сортировка внутри категорий — по близости к деньгам (стадия × сумма) и по срочности.

## Шаг 3 — формат вывода

1. «☀️ Брифинг на сегодня»
2. Цифры дня: открытый пайплайн $, сколько сделок, сколько просрочено по счетам $.
3. 🎯 **#1 на сегодня** — одно самое важное действие.
4. Топ-5–7 карточек: `Сделка — $сумма — стадия · почему сейчас · следующее действие`.
5. ⏳ Застряли.
6. 💸 Счета к оплате.
7. В конце — «С чего начнём?».

## Гейты безопасности

- Письма/сообщения — готовь только черновиками, никогда не отправляй сам.
- Правки в CRM — только с явного подтверждения.
- Сам брифинг — только чтение, ничего не меняет в данных.
